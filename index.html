⚔️ Slide 11: Konflik Antar Extension & Cara Import

// File: string_extensions.dart extension StringCaseExtensions on String { String capitalize() => /* ... */; } // File: text_utils.dart extension TextUtilsExtensions on String { String capitalize() => /* ... berbeda ... */; } // SOLUSI: import dengan prefix atau show/hide import 'string_extensions.dart' as se; import 'text_utils.dart' hide capitalize; void main() { var text = 'hello'; // text.capitalize(); // ERROR: ambiguous print(se.capitalize(text)); // Gunakan prefix }

✔️ Gunakan as prefix atau show/hide untuk menyelesaikan ambiguitas.

📌 Slide 13-14: Ringkasan & Best Practices

📖 Ringkasan:

  • Extension methods: menambah method ke tipe yang ada tanpa inheritance.
  • Sintaks: extension Nama on Type { ... }
  • Support method, getter, setter, operator.
  • Generic extensions: extension on List<T> sangat powerful.

⭐ Best Practices:

✅ Nama jelas: StringExtensions, DateTimeUtils
✅ Group by type: satu file per tipe utama
✅ Hindari overuse: jadikan semua utility sebagai extension jika masuk akal
✅ Null safety: handle nullable types dengan baik
✅ Dokumentasi: tambahkan komentar untuk public extensions
🎯 Extension methods membuat kode Dart lebih readable & maintainable. Gunakan dengan bijak.

🧠 Praktik Lanjutan & Catatan Penting

✨ Extension methods juga dapat digunakan untuk operator overloading pada tipe eksternal, serta memperkaya API library pihak ketiga.

📐 Contoh kecil: menambahkan operator * pada Duration? Atau getter isEven pada int? Semua mungkin dengan extension.

// Contoh extension dengan operator extension IntOperator on int { int operator [](int index) => this * index; // hanya demo kreatif bool get isEven => this % 2 == 0; } void main() { print(5.isEven); // false print(3[4]); // 12 (demo) }
